# Synthera Labs — Standards

Estándares de desarrollo, arquitectura, seguridad y documentación que rigen todos los proyectos de Synthera Labs.

Este repositorio es la fuente de verdad sobre cómo trabajamos. No es una guía de sugerencias, es el estándar que todo proyecto debe cumplir.

## Principios Fundamentales

Estos principios aplican a todos los proyectos, sin excepción:

- **Seguridad por diseño** — La seguridad se planea desde la arquitectura, no se agrega al final.
- **Documentación obligatoria** — Todo proyecto entregado debe incluir documentación mínima definida en `documentation/`.
- **Código limpio** — Nombres claros, sin duplicación, sin magia innecesaria.
- **Sin secretos en el código** — Toda credencial va en variables de entorno.
- **Revisión antes de merge** — Ningún cambio entra a `main` sin revisión.

---

## Cómo usar este repositorio

Antes de iniciar cualquier proyecto en Synthera Labs:

1. Lee los estándares de código del lenguaje que vas a usar.
2. Revisa los principios de arquitectura.
3. Completa el checklist de seguridad antes de hacer deploy.
4. Verifica que la documentación mínima esté presente antes de cerrar el proyecto.
